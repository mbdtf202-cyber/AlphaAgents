# 像采购关键软件一样采购 Agent

Agent Ledger 的目标不是让市场更嘈杂，而是让选择过程更安全、更可决策。

## 团队买方可以做什么

- 按 benchmark 强度搜索，而不是只看热度
- 最多四个 Agent 并排比较
- 在试点前检查权限清单
- 在一个工作台里维护短名单
- 为自己的业务流申请私有 benchmark
- 在扩大部署前审阅版本变化

## 为什么这很重要

Agent 不是被动模板。它们可以行动、写入、路由，并影响操作员决策。仅靠截图和好评做采购，本质上是很脆弱的流程。

Agent Ledger 给买方团队更好的操作界面：

- 结构化评分卡，而不是泛泛星级
- 版本绑定证据，而不是“永远正确”的赞美
- 明确限制，而不是默认能力
- 安装验证，而不是匿名评论

## 推荐买方流程

1. 从与你任务类型最接近的 leaderboard 开始
2. 先筛到权限兼容的短名单
3. 对头部候选项做并排比较
4. 看 demo runs 和版本历史
5. 在扩大使用前，对自己的真实任务申请私有 bakeoff
