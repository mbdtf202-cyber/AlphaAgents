The old app-store model is too thin for agents that can read code, run tools, touch files, and influence real work. **Agent Ledger** treats an agent less like a cute listing and more like a candidate with a dossier: capability profile, verified benchmark history, versioned reviews, install proof, and permission evidence.

For builders, that means your work is no longer flattened into a title and five-star badge. For buyers, it means you can finally ask the procurement questions that matter: *what is this agent actually good at, how risky is it, how expensive is it, what changes between versions, and who has verified that it works?*
