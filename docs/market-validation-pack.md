# AlphaAgents 市场验证与融资证据包

## 文档边界

本文件是市场验证、销售作战和融资材料取证清单，不是产品范围、视觉方案或工程契约。任何融资或销售表述都必须保留证据状态；没有真实付款、采购邮件、签约、客户授权交付包或验收记录时，不得把样例、访谈模板或沙盒包描述为真实商业验证。

## 1. 目的和证据边界

本文件把「能融资、能成交、能复购」需要的证据拆成可收集、可审计、可复盘的材料。

重要边界：

- 本文件中的账户、访谈和样单可作为脱敏样例或销售作战格式；未取得真实客户授权前，不得把样例包装成真实成交证据。
- 融资材料必须把证据状态标为 `validated`、`sandbox_verified`、`in_conversation`、`sample_only` 或 `target_to_collect`。
- 只有真实付款、真实采购邮件、真实 LOI、真实试单意向、真实交付包才能进入投资人材料的 `validated` 区。

证据分级：

| 等级 | 可用于 | 证据标准 |
| --- | --- | --- |
| `validated` | 投资人主材料、采购背书 | 付款记录、合同、采购邮件、LOI、客户授权的脱敏订单包、验收记录 |
| `sandbox_verified` | 产品演示、CI、盲审结构校验 | 样单工件可打开、事件序列可回放、财务账本和 UI/CLI/API snapshot 可校验；不代表真实客户 |
| `in_conversation` | 销售漏斗、PoC forecast | 有明确联系人、预算、验收人、下一步和预计时间 |
| `sample_only` | 销售演示、产品测试 | 明确标注为样例，不声称真实客户 |
| `target_to_collect` | BD 作战表 | 可触达账户但未访谈或未确认需求 |

| Standard/Pro | 0 | 0-1 | 2-4 | 5-8 | Trial 没复购或交付弱 |
| 企业 PoC | 0 | 0-1 | 1-2 | 2-3 | 采购包不足 |

| 年度 order-credit 谈判 | 0 | 0 | 0-1 | 1 | 缺 ROI 和 SLA 证据 |

每周复盘必须回答：

- 哪个 segment 回复率最高。
- 哪个反对意见最阻塞付款。
- 付款前最缺的材料是样单、SOW、DPA、发票说明还是 ROI。
- Trial 到 Standard 的升级理由是否可复述。
- 服务方交付是否能在不增加平台人工的情况下复用。

## 6. 单位经济敏感性表

基准假设：

- Standard 单价 6,800 CNY。
- 平台 take rate 25%。
- Provider payout 60%。
- Runtime/object storage 成本 3%。
- QA/ops 人工成本按 120 CNY/hour。
- CAC 按首单 1,200 CNY，复购订单不重复计 CAC。

| Scenario | Orders | Avg GMV | Take rate | Payout | QA/Ops hours | Refund/dispute cost | CAC/order | Contribution margin |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| Trial-heavy early | 10 | 2,944 | 22% | 62% | 1.4 | 6% | 900 | -8% |
| Standard learning | 25 | 5,820 | 25% | 60% | 1.1 | 4% | 520 | 13% |
| Repeat buyer base | 50 | 7,480 | 26% | 58% | 0.8 | 3% | 260 | 26% |
| Agency PoC mix | 80 | 8,900 | 27% | 57% | 0.7 | 3% | 180 | 32% |
| Enterprise credit | 120 | 10,500 | 28% | 55% | 0.55 | 2% | 120 | 40% |

敏感性结论：

- 毛利提升的第一杠杆是复购和 Standard/Pro 占比，而不是压低服务方 payout。
- QA/ops 小时必须从首单 1.4h 降到复购 0.7h 以下，否则规模化会被人工吞掉。
- 退款争议成本超过 6% 时，必须优先修验收模板、证据标准和供应准入，不扩市场。
- CAC 回收应在第二个 Standard 或第一个 Pro 订单内完成。

## 8. ROI 计算器

ROI 口径不能只靠首单直接人力替代。AlphaAgents 的购买理由是更短周期、证据可回看、交付责任、可复用选题、争议可裁决和复购后的边际成本下降。若首单 direct savings 为负，应作为定价和复购压力的风险提示，而不是销售主张。

销售演示用公式：

```text
internalCost = (operatorHours * operatorHourlyCost) + (managerHours * managerHourlyCost)
alphaAgentsCost = orderAmount
reviewCost = buyerReviewHours * managerHourlyCost
netSavings = internalCost - alphaAgentsCost - reviewCost
cycleTimeGain = originalCycleHours - alphaAgentsDeliveryHours
usableTopicCost = alphaAgentsCost / usableTopicCount
repeatDecision = acceptanceScore >= 85 and usableTopicRate >= 0.7 and reviewHours <= 2
```

Standard 示例：

| Field | Value |
| --- | ---: |
| Operator hours replaced | 12 |
| Operator hourly cost | 220 |
| Manager hours replaced | 3 |
| Manager hourly cost | 500 |
| Internal cost | 4,140 |
| AlphaAgents order | 6,800 |
| Buyer review hours | 1.5 |
| Review cost | 750 |
| Net direct savings | -3,410 |
| Cycle time gain | 72h -> 18h review-ready |
| Usable topic count | 23 |
| Cost per usable topic | 296 |

解释口径：

- Trial/Standard 不一定只靠直接人力成本打赢，关键是周期、证据、可复用选题和采购可追责。
- Agency 买方的 ROI 往往来自减少 account team 周报压力和提升客户交付稳定性。
- 年度 order-credit 的 ROI 必须看复购订单的边际成本下降，而不是首单。
- 首单 `netSavings < 0` 时，销售必须用 review-ready cycle gain、usable topic cost、复购边际成本和采购责任链解释价值；不能把负节省包装成省钱。

## 9. Validated evidence gap

当前仓库已有 3 份 `sample_only + sandbox_verified` 样单，用于证明 accepted、revision、dispute partial release 三条结构路径都可校验：

- `AA-SANDBOX-TRIAL-001`
- `AA-SANDBOX-REVISION-002`
- `AA-SANDBOX-DISPUTE-003`

它们仍然不构成 `validated` 商业证据。进入融资主材料前必须至少补齐以下任一组合：

| Evidence | Minimum acceptable proof |
| --- | --- |
| Paid Trial | 真实付款记录、订单 terms snapshot、验收人确认、可脱敏 evidence package |
| LOI / procurement email | 客户域名邮箱或签署文件，包含预算、使用场景、验收人和下一步时间 |
| Repeat buyer | 同一客户 3 单订单记录、验收记录、复购原因和退款/争议状态 |
| Enterprise PoC | 采购/法务/财务联系人、SOW redline、付款计划或合同审批截图 |

没有上述材料时，对投资人的表述必须是：产品与证据包结构已经 `sandbox_verified`，商业牵引仍处于 `target_to_collect` 或 `in_conversation`。

- 已有 3 份 `sample_only + sandbox_verified` 样单，可证明 accepted、revision、dispute 三条路径上的证据包、事件序列、QA、财务账本、声誉回写和 UI/CLI/API snapshot 结构能被校验。
- 已有 90 天销售作战表、访谈记录模板、漏斗目标、单位经济敏感性和 ROI 测量口径。

当前不能对外声称：

- 不能声称已有真实付费客户、真实复购、真实 LOI、真实采购邮件或真实年约谈判。
- 不能把 30 个目标账户和 10 个访谈样例作为已经发生的客户事实。
- 不能把 sandbox ledger、sample ROI 或 synthetic evidence package 放进融资材料的 `validated` 区。
- 不能宣称 enterprise procurement ready，直到真实签约主体、收款主体、开票主体、退款路径和子处理方清单填实。

## 10. 投资人材料必须输出

| Artifact | Minimum bar |
| --- | --- |
